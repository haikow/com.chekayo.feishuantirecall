# 飞书防撤回 (Feishu Anti-Recall) · LSPosed 模块

在**原生存储层**拦截撤回，让被撤回的消息原文继续显示 —— **包括你从未打开过的聊天**。

> ⚠️ 本仓库**仅提供编译好的 APK 下载，不包含源码**。仅供学习研究与个人使用，使用风险自负。

---

## ✨ 特点

- **原生 SQL 层拦截**：hook `libsqlcipher.so` 的 `sqlite3_step`，在"撤回写入"落库前把它改绑到废弃主键，原始消息行原封不动保留。
- **覆盖未打开的聊天**：原文在收信时就已写入本地库，不依赖聊天是否打开 —— 这是纯 Java/UI 方案做不到的。
- **免电脑常驻**：装好启用后随飞书启动自动生效，**不需要连电脑、不需要 frida**。
- **较稳**：只认数据库表名/列名和 SQLite 标准导出函数，不依赖会随版本变化的业务函数地址。

## 📦 支持范围

| 项 | 说明 |
|---|---|
| 架构 | **仅 arm64-v8a**（覆盖 2017 年后几乎所有真机；armv7/x86 暂不支持） |
| 系统 | Android（需 Root + LSPosed / 兼容的 Xposed 框架） |
| 飞书 | 在 `com.ss.android.lark` **7.52.4** 上验证；其它版本通常可用，若失效需重新适配 |

## 🔧 安装

1. 在 [Releases](../../releases) 下载最新 `feishu-antirecall-*-arm64.apk` 并安装。
2. 打开 **LSPosed** → 模块 → 启用「飞书防撤回」。
3. 在该模块的**作用域**里勾选 **飞书 (`com.ss.android.lark`)**。
4. **强制停止飞书**再重新打开（设置 → 应用 → 飞书 → 强行停止），让模块加载。

> 校验是否生效：`adb logcat -s antirecall-native` 出现
> `ANTI-RECALL native hook installed on sqlite3_step` 即已挂上；撤回发生时会打印 `NEUTRALIZED recall write`。

## 📖 使用说明 / 已知局限

- ✅ **模块生效后**被撤回的消息，打开聊天即可看到原文。
- ℹ️ 撤回时飞书另插的「**X 撤回了一条消息**」系统提示行仍会显示，当作"这里发生过撤回"的天然标记。
- ❌ **模块生效之前**就被撤回的旧消息，原文已从本地库清除，**无法找回**。
- ℹ️ 打开聊天时飞书的对账可能短暂多次重试撤回（均被拦下，很快收敛，不影响使用）。

## ❓ 失效了怎么办

飞书大版本更新后若不生效，多半是表结构或加载方式变化，需要重新适配（重新核对 `messages` 表列序与 `is_recalled` 位置）。可在 Issues 反馈飞书版本号。
